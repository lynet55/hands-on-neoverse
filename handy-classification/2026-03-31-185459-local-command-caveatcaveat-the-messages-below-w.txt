  The dataset (data.py) returns {"image": tensor, "ground_truth_image": tensor}

  4) Training loop logic review

  The overall structure (reconstruct ‚Üí classify ‚Üí loss ‚Üí backprop) is sound,
  but:

  - Gradient flow is broken: NeoVerseReconstructor runs under torch.no_grad in
  app.py and uses torch.amp.autocast. The reconstruction is non-differentiable
  (GS rasterization via gsplat). loss.backward() will only update the
  ClassificationHead, not the reconstructor. This is fine if intended (frozen
  backbone), but should be explicit with torch.no_grad() around the reconstruct
  call.
  - Loss function: CrossEntropyLoss expects integer class labels or
  probabilities, but gt_mask is a segmentation image tensor. You'd need to
  convert the mask to class indices first.
  - Logits shape vs target shape: The classification head outputs [B, S, 
  num_classes] (per-frame class logits). The ground truth mask is a spatial
  image. These shapes don't align for CrossEntropyLoss. You likely want either
  pixel-level segmentation (needs a different head) or a single class label per
  sample.
  - batch_size=16 in DataLoader vs cfg.batch_size=2: The config value is never
  used.

‚óŹ The answer is straightforward:

  You can feed rendered RGB directly into CrossEntropyLoss against segmentation 
  masks ‚ÄĒ but ClassificationHead can't do that. The head pools spatially
  (tokens.mean(dim=2)) and outputs [B, S, num_classes], collapsing all spatial
  information. You can't compare a per-image class prediction against a
  per-pixel segmentation mask.

  For pixel-level comparison against segmented ground truth, you need a dense 
  prediction head, not a pooled classifier. The worldmirror backbone already has
   exactly this ‚ÄĒ DPTHead takes the VGGT token list List[torch.Tensor] of shape
  [B, S, patches, 2*embed_dim] and outputs dense [B, S, C, H, W] predictions via
   multi-scale feature fusion. That's the right architecture for segmentation.

  So: using the VGGT tokens with a DPT-style head is preferred, because:
  1. The backbone tokens are already computed during reconstruction (no extra
  forward pass)
  2. DPTHead preserves spatial resolution ‚ÄĒ you get per-pixel predictions
  comparable to your GT masks
  3. Rasterized RGB alone has no learned features; you'd need another backbone
  to extract them anyway, duplicating work

  The rendered RGB is still useful ‚ÄĒ as a visual sanity check or as additional
  input ‚ÄĒ but the classification/segmentation signal should come from the VGGT
  tokens that worldmirror already exposes.
